Markdown

Minimal implementation of the Markdown format

Introduction

ReporteRs usage a too verbose when it has to handle lot of textual output. Also, addParagraph is less human readable than Markdown.

To make easier to let report text, markdown has been implemented in ReporteRs.

Details

The markdown definition used is John Gruber's one, documented here: DaringFireball.

Pandoc footnotes have been added. See pandoc README.

The function is to be improved, few things are still to be implemented:

underscores do not emphasis text (asterisks only are supported).
code blocks are expected to be valid R code
inline images are not available

The function does not implement HTML parsing but this is not in the todo list.

Example

The following script is producing the file markdown.docx

# set default font size to 10
options( "ReporteRs-fontsize" = 10 )

mydoc = docx( )
mkd = [1273 chars quoted with '"']





































mydoc = addMarkdown( mydoc, text = mkd,
        default.par.properties = parProperties(text.align = "justify",
                padding.left = 0) )

# Write the object
writeDoc( mydoc, file = "examples/markdown.docx" )
